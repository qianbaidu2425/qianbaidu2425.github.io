读《桶川跟踪狂杀人事件有感》
2023/11/24

这几天正好道法课大作业的ddl将近，同学推荐了一本书《桶川跟踪狂杀人事件》，就着这本书把大作业写掉了。

背景提要

这是有关一个女孩被人杀害了三次的故事，第一次是凶手，第二次是媒体，第三次是警方。

1999年10月26日，年轻女子猪野诗织在日本埼玉县JR桶川站前遭人刺杀。任职于新潮社《FOCUS》周刊的记者清水洁像往常一样展开了对该案的调查。然而，随着调查的深入，清水洁逐渐发现了这起案件背后种种骇人听闻的真相和令人发指的黑幕。经过长达四年的不懈努力，这起案件才逐渐尘埃落定。清水先生将他追踪此案的全过程记录在《桶川跟踪狂杀人案》一书中。

世人从来向往“天下大同”的理想社会。桶川一案揭露出的残酷现实，表明建设和谐社会仍道阻且长，需要但笔者认为清水先生的这本著作能为迷茫中的我们指引一些正确的方向。

社会秩序的维持离不开法治

猪野诗织第一次是被凶手杀害的。

诗织第一次遇见小松和人时，小松和人自称他是“从事汽车销售的青年实业家”，但是小松和人实际上经营着非法色情行业，在清水记者不懈的追踪下，他发现小松和人经营的色情按摩店规模相当大。

难以想象，在即将迎来二十一新世纪的时候，这些非法产业却在日本的社会上横行霸道。事实上，早在1900年，日本就出台了《淫行禁止法》，并在1951年出台了《淫秽物贩卖等罪》这一部法律。那么，这或许说明，当时的日本池袋地区的治安部门，或许根本就没有将禁止色情产业的法律坚决实行。：

同时，在当时的日本，并没有相关法律对跟踪骚扰行为做出规定。因此，就算是遇到骚扰以至于生命受到威胁的公民，也无法指望警署对骚扰行为采取措施，书中原话如下[1]：

“跟踪狂的问题，或许警方也难以判断。实际上，警方收到这类咨询求助的数量相当多。如果被害的一方有过错，反过来遭到警方斥责，或许是活该；但诗织的案例又如何？”

一个社会缺失了法律，就缺失了评判行为的准绳。正是因为没有相关法律的规定，小松和人才可以肆无忌惮地对诗织展开跟踪和骚扰，正是因为没有相关法律的规定，才使得诗织在向警方求助时，得到的是一句句冰冷的拒绝。2000年5月18日，日本通过了《跟踪骚扰行为规范法》，那些饱受跟踪和骚扰折磨的日本公民终于拥有了捍卫自己权利的途径。

一个好的社会是有序的，而秩序的维持离不开法律的约束和执法者的尽责。在未来，我们所处的社会还会出现更多游走于“灰色地带”的行为，如果不及时制定相关的法律并坚决贯彻法治的理念，那么这些行为就有可能酿成大错。

我们需要怎样的媒体环境？

猪野诗织第二次是被媒体杀害的。

在本书中，清水先生展现了当时日本媒体传播界的扭曲。在当时的日本，民众对《FOCUS》这样的周刊的印象往往是“三流”，而某些报纸却能被冠以“一流杂志”之名。如此巨大的待遇差距，不是因为它们报道的内容存在深度上的优劣之分，而是因为——周刊不是政府宣传型的“公共媒体”。

这样的成见带来的危害，在桶川一案中被放大了。办案的搜查员提起诗织一案时，常常这么说——“那是酒家女的三流案子啦。“警方用这样不合实际的话语对媒体进行了诱导，而那些所谓的“一流杂志”只会把政府机关公布的信息照本宣科地报道出来，并把这样的形容词放大后，插在标题里，放在版面上。如此一来，诗织的形象变成了“酒店小姐”、“迷恋名牌的人”活在了许多大众心中，但她只是一个普通的女生啊……

在如今的中国社会，我们也应该避免这种情况的出现。媒体的高低应该根据其发表的内容的质量来评判，国家应当在全社会进行这种价值导向的引领。同时，新闻工作者不应该随波逐流，应该要对信息来源采取客观的态度进行审视，敢于质疑，不能因为是“官方的”就不加思考地大肆报道。

同时，笔者认为，如今的媒体不应做“标题党“。显眼的标题会让读者先入为主，错误的信息可能会存在于读者的潜意识里，潜移默化地改变大众舆论的风向。同时，笔者也认为，国家应当进一步完善审查体制，筛查出那些准确而客观的报道，不让错误的信息流入社会。

国家机关工作人员的不作为是可怕的

猪野诗织第三次诗被警方杀害的。

在本书中，最令人感到愤慨的，或许就是埼玉县上尾警署的黑暗。他们怠政，只是因为“不愿意引起麻烦”就对诗织的求助报以冷漠的态度，随意篡改笔录；他们拥有数百人组成的专案组，侦察效率却不如周刊的几位记者。在办案的过程中，他们始终绕开最重要的环节——小松和人，直到最后他们都在逼迫小松和人的哥哥小松武史承认自己是主谋，而小松和人直到最后都仅仅止于被以名誉损毁的罪嫌通缉。最可恨的是——他们把死者发生的不幸当作儿戏。他们随意扣押死者的遗物，在死者父母接到死讯之后还蛮横地要求他们配合填写各类文件，甚至还将死者生前写下的日记里的某些片段拿来咬文嚼字，拿来反击死者父母的质疑。在办案的发布会现场，有警察甚至还带着笑容，实在令人作呕。

上尾警署的警员们，不但没有拯救诗织，放任她的生命被凶手夺取，还在诗织去世后不断践踏着她的人格。
书中描写的上尉警署，是日本当时无数不作为的警署中的一个缩影，或许也是现在中国社会中许多不作为的地方政府机关的一个缩影。这些不作为的官员，在他们的恶行被公之于众之前，他们仍然在日复一日地懒政、怠政；难以想象多少人民的权益因此而收到损害。

笔者认为，我们必须进一步加大反腐的力度。国家监察机关必须深入基层，推动反腐工作的展开。同时，要给民众提供更多的反馈渠道，让更多不作为、不担责的行为被曝光出来。在全社会营造廉洁而公正的行政氛围。

总结

笔者认为，我们应该感谢清水洁先生，是他让这起拥有里程碑式意义的案件的真相能够被世人所看见。或许，我们应该感谢所有怀揣良知、正义与勇气，为了真相而奋不顾身的人们。作为后人，我们能从这本书中获得灵感，让清水先生所做出的努力化为我们建设和谐社会的良策。

xpath的基本使用
2023/11/12

xpath的基本语法

1.路径查询

//：查找所有子节点，不考虑层级关系
/：找直接子节点

2.谓词查询

//div[@id] 查找带有某个id的节点
//div[@id = “…”] 查找带有某个id且id名称限定的节点

3.属性查询

//@class
注意这里返回的是class的属性（也就是class后面的那个字符串），无法跟text()连在一起用！！

4.模糊查询

//div[contains(@id, “…”)] 包含
//div[starts-with(@id,”…”)] 以某个字符开头的

5.内容查询

//div/…/text()

6.逻辑运算

//div[@id=”…” and @class=”…”]
//div[@id=”…” or @class=”…”]

xpath的两种解析方式

第一种方式：解析本地文件

<body>
<ul>
    <li id='l1' class='c1'>北京</li>
    <li id='l2'>上海</li>
    <li id='k3'>深圳</li>

</ul>
<ul>
    <li>厦门</li>
    <li>福州</li>
</ul>
</body>

from lxml import etree
tree = etree.parse("xpath的基本使用.html")#传入文件的名称

#查找ul下面的li
li_list = tree.xpath('//body/ul/li')#这里还有一种写法是：'//body//li'

#查找所有有id属性的li标签
li_list = tree.xpath('//body/ul/li[@id]')
li_list = tree.xpath('//body/ul/li[@id]/text()')#加上一个text可以获取标签中的内容——北京、上海

#查找id为l1的标签
li_list = tree.xpath('//body/ul/li[@id="l1"]/text()')#注意引号！

#查找到id为l1的li标签的class的属性值
li_list = tree.xpath('//body/ul/li[@id="l1"]/@class')

#查询id中包含l的li标签
li_list = tree.xpath('//body/ul/li[contains(@id,"l")]')

#查询id中以l开头的li标签
li_list = tree.xpath('//body/ul/li[starts-with(@id,"l")]')

#查询id为l1且class为c1的
li_list = tree.xpath('//ul/li[@id="l1"and @class="c1"] ')

#查询id为l1或id为l2的
li_list = tree.xpath('//ul/li[@id="l1"]/text()| //ul/li[@id="l2"]/text()')

注，以上所有的返回的li_list都是一个列表！只有属性查询不能接text(),其他都可以使用text()来显示标签中的内容

urllib库（8）代理和代理池
2023/11/6

代理和代理池

一个伪装的ip地址，用一个字典表示
例：

proxies = {
    'http':'118.24.219.151:16817'
}

代理池就是多个代理组成的字典
例：

proxies_pool = [
    {'http':'118.24.219.151:16817'},
    {'http':'113.124.93.38:9999'}
]

代理的使用语法

request = urllib.request.Request(url=url,headers=headers)
handler = urllib.request.ProxyHandler(proxies=proxies)
opener = urllib.request.build_opener(handler)
response = opener.open(request)

从代理池中随机抽取一个代理：

import random
proxies = random.choice(proxies_pool)

urllib库（7）异常
2023/11/6

常出现的两种异常：

1、HTTPError：出现在url种某个字符出现问题的时候
2、URLError:出现在这个url不存在的时候
其中：HTTPError是URLError的一个子类

示例：

import urllib.request
import urllib.error
url1 = 'https://blog.csdn.net/boysoft2002/article/details/132520234'
url2 = 'https://blog.csdn.net/boy__soft2002/article/details/132520234'#HTTPError
url3 = 'http://www.yangkeyu.com'#URLError



headers = {
'Accept':
'text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.7',
'User-Agent':
'Mozilla/5.0 (Linux; Android 6.0; Nexus 5 Build/MRA58N) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/118.0.0.0 Mobile Safari/537.36 Edg/118.0.2088.46'
}


try:
    request = urllib.request.Request(url=url3, headers=headers)
    response = urllib.request.urlopen(request)
    content = response.read().decode('utf-8')
    print(content)
except urllib.error.HTTPError:
    print('出现http错误')
except urllib.error.URLError:
    print("出现了url错误")
